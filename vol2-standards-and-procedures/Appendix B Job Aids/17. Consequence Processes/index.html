    Job Aid 17—Consequence Processes¶

    Background¶

    This document describes processes for acquisition of the structure inventory, adjustments to the structure inventory, and computation of consequences for a Future of Flood Risk Data (FFRD) project.

    Preparing Structure Inventories (Data Preparation Phase)¶

    The delivery process for the initial structure inventory for an FFRD project during the data preparation phase is described below.

    Structure Inventory Scales¶

    The structure inventory points are represented by geopackage files using the U.S. Army Corps of Engineers (USACE) National Structure Inventory (NSI) data schema documentation. There are two scales of structure inventory used in FFRD. The first is the Hydrologic Unit Code (HUC)-4 watershed scale, where all the structures in the basin are in one file. The second scale is at the modeling unit boundaries (the River Analysis System [HEC-RAS] model boundaries), where the HUC-4 level file is split into one geopackage for each modeling unit. These smaller scale files cannot be created until the modeling unit boundaries have been finalized. This job aid describes how to create the structure inventories using the NSI data as the source.

    Structure Inventory File Type and Naming Conventions¶

    Each structure inventory file is a geopackage containing a point data layer named nsi as the only table. The name of the geopackage itself is defined in Job Aid 06, Standard Naming Conventions.

    Full Watershed Study Structure Inventory¶

    During the Scoping and Data Preparation Phase, prior to the creation of the mesh refinement regions, the full study watershed structure inventory is developed. There are two primary methods to download the NSI data. The first is directly from the NSI website. This allows for downloading of NSI geopackages at the state level. Clip the state geopackage data to the study watershed boundary, and if multiple states are required, clip each state file to the watershed, and merge the output files into a single geopackage for the entire watershed.

    The second method is to use the county-level NSI api to download county-level data in JSON format, convert the files to geopackage, clip the data to the watershed boundary, and merge the outputs together to a single output geopackage. This can be accomplished using a python script tool in a geographic information system (GIS) environment.

    Delivery Location: basin-name > basin-data > structure-inventory

    Modeling Unit Structure Inventories¶

    Once the modeling unit boundaries are finalized, the full watershed structure inventory geopackage is split into separate geopackages for each hydraulic model unit. In ESRI software, the split tool is used, and the split results are used to create the unique geopackage files for each modeling unit. This can be accomplished more efficiently through a script that will pre-form the split and create the individual geopackage outputs with the appropriate naming conventions.

    This dataset will remain without further refinements and serve as a reference to the original NSI dataset throughout the project.

    Delivery Location: basin-name > basin-data > structure-inventory

    Improvements to Structure Inventory (Calibration Phase)¶

    The purpose of this section is to identify potential issues in the hydraulic modeling and the placement of the structure inventory points. This subsection applies to calibration phase and occurs after the structure inventory data is prepared.

    Identifying Hydraulic Modeling and Structure Placement Issues¶

    Improvements to the structure inventory can include resolving structure placement issues.

    Workflow and Timeline¶

    The following ordered steps occur during the calibration phase of the FFRD project.

    1. Once the HEC-RAS model units have been finalized and the HEC-RAS meshes have been developed, the consequences team member will generate structure inventories for each modeling unit.

    2. The hydraulic modeler utilizes the inventory by computing depths for the calibration events at the structures and reviewing the depths for outliers.

    3. The hydraulic modeler will resolve any hydraulic modeling issues identified and compile any observed issues with the structure inventory that the consequences team member should review.

    4. Once a hydraulic model is submitted for review with computed depths at the structures for each calibration event, the consequences team member will pull the files locally for review.

    5. The consequences team member will review the depth estimates for the events and the issues identified by the hydraulic modeler to review and identify issues that may be hydraulic in nature or structures that need to be relocated or removed. The consequences team member will provide a list of potential hydraulic modeling issues back to the hydraulic modeler and update the structure inventory as required to address structure inventory location issues.

    6. The consequences team member will deliver the calibrated structure inventory project directory.

    Delivery Location: basin-name > calibration > consequences

    1. As described in Job Aid 07, Model Library Use, the current dataset to be applied for computations must always maintain a consistent pathname and filename. When iterations of the structure inventory improvements occur, outdated versions may be archived by moving them to an archive subdirectory with an appended date stamp included in the filename.

    Improvements to Hydraulic Model or Structure Placements¶

    Reviewing hydraulic model outputs with the structure inventory helps to focus the review to areas that are currently high risk and will receive greater scrutiny as well as to provide an additional perspective in the review aside from gage-based metrics alone. The hydraulic team members and the consequences team members complete this job aid. The following sections provide some context for common issues in hydraulic modeling and structure inventory placements. The hydraulic modeling team member will focus on identification of hydraulic modeling issues during calibration of the hydraulic models; however, it is likely that in reviewing the results some structure placement issues may be noticed and documented. The consequences team member is responsible for structure placement issues; however, it is likely that the consequences team member might notice hydraulic issues missed by the hydraulic modelers during their review.

    Hydraulic Model Refinements¶

    Hydraulic model refinements are necessary when the hydraulic model results in depths at structures that are unreasonable for the modeled event. This can happen due to lacking terrain modifications, poorly aligned cells, datum issues with the levee crest profiles, or other geometry or plan settings. Some common issues are illustrated below.

    Terrain Modifications¶

    When defining river channels in hydraulic models, sometimes islands containing infrastructure can be improperly defined in the channel. Simply reviewing the hydraulic modeling without noticing the structures can yield hydraulic model results that look good, but are actually inconsistent with reality. Figure 1 illustrates an example where an industrial complex on an island floods with every calibration event even though the complex is rarely flooded. The terrain did not reflect the island adequately.

    Figure 1. An Incorrectly Flooded Industrial Complex due to Terrain

    Stream Alignment Issues¶

    If automated tools are used to define stream centerlines, or old orthophotography is utilized to review a model, it can happen that a river does not align with the real river. Using the structure inventory can quickly identify these issues.

    Structure Inventory Refinements¶

    The base level NSI utilizes parcel data and building footprints to locate structures. No single parcel data source in the United States is fully complete and no footprint data source has yet achieved a perfect representation of all buildings in the United States. The generation of the NSI leverages other sources of information to fill in data gaps. An example is reliance on the Census data census block level housing units with other Census datasets to generate structures when no parcel or footprint data exists. Reconciliation of these diverse datasets can yield incorrect results occasionally. It is advisable that manual calibration of the inventory is performed to prevent overestimation of average annual damages. The consequence team member is responsible for calibration of the inventory used for production of results. The following sections provide some low effort solutions to improve significant issues quickly. The NSI technical documentation Frequently Asked Questions can also provide background on common issues and how to address them. The primary objective of this job aid is structure placement improvements or significant issues impacting consequence estimation.

    Initial Depth Outputs Sort By Depth¶

    Calibration using the computed results from the hydraulic modeling calibration can quickly identify structures that are in areas that have previously flooded. In some cases, the structures are inappropriately placed and should be relocated. Common issues occur near rivers when no building footprints exist, but a parcel record does. By default, the NSI drops a point in the center of the parcel, which is good for a neighborhood, but some states extend parcels to the centers of rivers, which can lead to structures being placed in the centers of rivers. By sorting on depth and reviewing the deepest depths at structures the inappropriately placed structures can be quickly reviewed and relocated. Figure 8 illustrates an example of deep depths on a nonexistent structure identified in a parcel for a potential harbor.

    Figure 8. Nonexistent Structure in a Harbor

    Location Based on Imagery¶

    Various imagery sources are used in GIS software to adjust and verify structure locations. This effort focuses on areas near water sources such as rivers, creeks, streams, and lakes. Existing data, i.e., flood maps and dam and levee breach events, are used to target areas for location calibration. A primary goal of this calibration effort is to ensure structure points are not placed in areas subject to continuous inundation, such as within a river channel.     Improvements to Structure Inventory (Conformance phase)¶

    Evaluation of the quality of the hydraulic modeling and the placement of the structure inventory continues during the Conformance Phase of the project. The Conformance Phase starts with a calibrated structure inventory coming from Calibration Phase and hydraulic results that span the frequency range from the 5-year to the 2,000-year event become available to aid in the review of hydraulic model and structure placements.

    Conformance Phase Structure Inventory Data Management¶

    At the start of the Conformance Phase, a copy of the Calibration Phase structure inventory is promoted to the conformance phase directory location. Any updates that to the inventory during the phase will be maintained with the same path and filename, and outdated versions may be archived.

    Structure Inventory Location for Conformance phase:

    basin-name > conformance > consequences

    Evaluation of Damages in Frequent Events¶

    The results from hydraulic and consequence modeling in the Conformance Phase are obtained as geopackages from the model library and reviewed to identify potential issues. Any structures flooded during the most frequent event (typically the 2-year event) are checked for accuracy using aerial imagery and other sources as described in Improvements to Structure Inventory (Calibration Phase). Any potentially significant issues with hydraulic modeling will be discussed by the team. Structure point locations are edited when necessary to reflect realistic results. Examples of this include moving a point from the center of a structure to an edge to keep it from flooding at the most frequent events.

    Evaluation of Damages and Flood Frequencies in Leveed Areas¶

    Results within leveed areas are inspected to determine if flood frequencies and damages on the interior of the levee are higher than expected. Some water will likely pool in a levee interior during frequent events, but there shouldn’t be large amounts of structure flooding at frequent events (10-year and more frequent). The Annual Exceedance Probability grid can be used in conjunction with the other results files to inspect levee system results for reasonableness.

    Any significant issues with interior levee results will be documented and reported to discussed with the team for further investigation. Any issues not addressed will be documented in the reporting, including any identified causes and solutions. Examples of remaining issues could be levees that do not tie into high ground or lack of pumping and interior drainage systems in the hydraulic model.

    Computation of Consequences¶

    Technical procedures for computation of consequences based on hydraulic modeling results are included in cloud-compute plugin documentation. In general, the inputs to a consequence computation are a set of hydraulic model results describing the hazard, a structure inventory describing the exposure, and a set of damage functions describing the vulnerability. The FFRD directory structure described throughout this SOP support the standardized delivery of input data for the consequence compute.

    Production Phase Structure Inventory¶

    At the start of the production phase, a copy of the conformance phase structure inventory is promoted to the production phase directory location.

    Structure Inventory Location for Conformance phase:

    basin-name > production > consequences
