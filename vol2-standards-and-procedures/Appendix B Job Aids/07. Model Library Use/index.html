    Job AId 7—Model Library Use¶

    This document defines key details regarding the use of Model Library specifically for the FFRD initiative. The Model Library application allows access to view and interact with datasets stored in the cloud, independent of the specific bucket, and meets all USACE security protocols. The Model Library supports the FFRD program by providing a place for team members to store, share, and review models as well as providing a platform for staging models for cloud compute.

    Model Library Key Information¶

    The Model Library is a general tool used for projects other than FFRD. Key background information can be found via the links below.

    Current Model Location¶

    FFRD projects have many team members contributing throughout the process and various models and datasets will be updated multiple times throughout their development. Strong control of model versions is critical to avoid potential errors of oversights.

    The key tenant of FFRD model data management is that the current version of each model will always remain in a single, stable location.

    Archive Process for Superseded Model Data¶

    To maintain current model data in a stable location, user action to intentionally archive the current model prior to replacing it with updated version is required. Within the ‘calibration’ and ‘conformance’ directories, each model has its own subdirectory. The ‘archive’ directory will store the older models as they become out-of-date. Older models will be moved to the archive directory using the rename action on the subfolder level and entering, “archive\model-name-YYYYMMDD” as the location to move to. This move process is described at the Move a File or Directory, Model Library user’s manual page.

    User Account Management Actions¶

    The following subsections provide common user account management actions for FFRD.

    Team Members: Gaining Access to Model Library¶

    The process for gaining access to the Model Library is initiated by entering the URL (https://model-library.sec.usace.army.mil/) into a web browser. The general workflow for creating an account is included in the Model Library documentation, which can be accessed from the help button (Figure 2) in the upper right corner of the page.

    Figure 2. Model Library Help Button

    Once access is gained, legitimate user verification by the Model Library administrators may be necessary. If access has not been granted, inquire with the help desk via email at mlhelp@usace.army.mil.

    After new account verification is complete, users will need to be added to the FFRD modeling group in Model Library. Reach out the team lead if access is needed.

    Technical Lead: Grant Group Access to Team Members¶

    The technical lead will grant access for team members to the project repository. The technical lead must be already added to the “FEMA-FFRD” group and assigned as a group administrator by the Model Library administrators.

    To add a team member to the group:

    1. Select the drop-down button on the upper right of the browser

    2. Select “Manage Groups”

    3. Select the FEMA-FFRD card

    4. Select the Select new user drop down button and find team member

    5. Select blue edit button next to new team member

    6. Apply the appropriate permissions (Read and Write for most team members)

    7. Save

    8. Verify user has access via offline communication

    If any technical issues or questions arise, support can be attained via email at mlhelp@usace.army.mil.

    Technical Lead: Create New FFRD Repository¶

    At the beginning of an FFRD project a new repository will be created in the Model Library by the technical lead, and empty folders will be created according to the standard directory structure.

    An FFRD demo repository is available in the Model Library to aid in the creation of a new project repository. This demo repository is named, ‘ffrd-folder-structure-demo’ and includes all of the directories and sub directories needed for a new repository. Note that additional basin name specific folders must be added and read-me files are included in the demo directory to guide the format for those folders.

    To create a new repository for a new project, follow these steps:

    1. Download the demo directory to your computer

    1. Unzip the downloaded directory

    2. Add custom, basin-specific subdirectories as indicated in the read-me files

    3. Click Add a Model Repository from the Model Library Splash Page

    1. Use the FFRD SOP standard naming for the repository (e.g., ‘ffrd-trinity’)

    2. Set Program field to “FFRD”

    3. Set Repository Group to “FFRD”

    4. Set Repository Location to “FFRD-Computable”

    5. Set Repository Purpose to “Watershed”

    1. Open newly created repository

    2. Click on “Tools – Upload Files”

    3. Drag the local directory from your computer into the upload tray in the browser.
