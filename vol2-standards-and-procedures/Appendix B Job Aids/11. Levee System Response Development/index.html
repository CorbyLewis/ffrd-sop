    Job Aid 11—Levee System Response Development¶

    This job aid describes the procedures to collect levee performance data, scope the levee breach modeling methodology, produce system response inputs, and communicate with the project team to incorporate levee breaches into the modeling system.

    Context¶

    To incorporate levee performance into FFRD, one or more breach locations must be identified, system response probability (SRP) curve(s) must be estimated, and a breach scenario must be incorporated into the hydraulic and hydrologic models and cloud-compute inputs.

    Roles¶

    The following key roles are required to incorporate levee breaches into an FFRD project.

    Levee Data Lead¶

    The levee data lead is the USACE expert responsible for the development and delivery of SRP curves for levees. They also contribute to levee database refinement efforts. Additionally, they review and evaluate breach results from stochastic model simulations.

    Hydraulic Modeler¶

    The hydraulic modeler is responsible for developing the model that fully accommodates levee breaches as scoped. Additionally, the hydraulic modeler confirms the levee control location (LCL) (initial overtopping locations) for each levee system based on modeled water surface profiles.

    Cloud Compute Lead¶

    The cloud compute lead incorporates hydraulic and hydrologic models and SRP relationships into cloud compute for the project and ensures that results are shared with the team for review.

    Procedural Overview¶

    Establishing levee two-dimensional (2D) structures, breach locations, and SRP curves for FFRD modeling have the following main steps:

    1. Prior to FFRD Study: Refine and improve the quality of the National Levee Database within the basin prior to the FFRD project. Guidelines for these separate efforts are not included in this SOP. This effort includes improvement to the levee alignments, features, profiles, and leveed areas.

    2. Scoping and Data Preparation Phase:

    1. Identify which levee systems/segments are included in the study area and what levee data is available.

    2. Consolidate and evaluate existing levee risk assessment data

    3. Select number of breach location for each levee system

    4. Identify and document preliminary LCL breach locations

    5. Identify levee breach quantity and location(s) per guidance. Data is needed when Calibration Phase starts. Note that the breach location—particularly the LCL—may be adjusted during Calibration Phase.

    1. Calibration Phase:
    1. Verify and finalize breach locations, including assessment of LCL locations with the support of hydraulic model results

    1. Define levee breach sampling scheme

    2. Develop SRP curves for each breach location and upload to project directory

    1. Conformance Phase: Review and evaluate resulting sampled breach elevations, and number of breach events that occur within the single realization conformance phase computation. Address any issues.

    2. Production Phase: Review and evaluate breach results from multi-realization event computations.

    3. Final Reporting Phase: Ensure proper documentation of breach inputs and results. Note that interim reporting is integrated into each FFRD project phase, and the end-of-phase reports will provide much of the technical documentation of levee breach inputs and results.

    Identifying Levees in the Study Area¶

    The National Levee Database will be used as the authoritative listing of levee systems in the study basin, and every system will include at least one breach location for the FFRD project. NLD data refinement efforts that occur prior to the start of the FFRD project will ensure that the levee inventory is of high quality for project use. The Levee Data Tracking table will provide insight on the previously completed data refinement activities.

    Levee Breach Quantity and Location Selection¶

    Breach location selection and quantity is a critically important but challenging aspect of levee modeling. The location and number of breaches impacts the potential inundation area in the FFRD results.

    The levee data lead determines the breach quantity. The breach quantity will be set by a combination of many factors including levee mileage, number of levee segments, and risk assessment data availability. The breach quantity can be adjusted based on engineering judgment. The breach location will follow a general hierarchy for most critical areas to define but will also rely on engineering judgment in some cases. Documentation of applied judgement is required (See Subsection 1.5.1).

    The following subsections outline the procedures. Selecting breach quantity and location can be expected to be iterative with the HEC-RAS modeling team; particularly with identifying a LCL, which is the location on the levee system expected to experience overtopping first. The initial breach locations are needed by the hydraulic modelers ahead of the calibration phase. The calibration model runs may yield a different understanding of the LCL location that will need to be adjusted.

    Levee Breach Quantity¶

    Breach quantity is a balance of the level of detail required to display a representative picture of flooding and the inundation characteristics of the levee. Long levee reaches, levee systems with steep terrain, and/or short hydrograph floods present a theoretical need to increase the number of breach locations to provide a more comprehensive breach inundation. For example, a breach location at the downstream end of a long levee system will not incur inundation to the upper reaches of the leveed area. If available information indicates this is the most likely scenario for levee performance to contribute to the system, it is acceptable. In the absence of risk assessment data, there is uncertainty in making the selection.

    When using levee risk assessment information, the breach quantity will be scaled to accurately represent what is known about the levee and not mischaracterize levee performance by over manipulating the input data. For example, combining multiple system responses from different Potential failure modes (PFMs) and locating them at a LCL or frequently loaded location will increase the annual probability more than intended. Similarly, arbitrarily splitting input data to apply to a larger number of breach locations may result in levee performance mischaracterization by the modeler and increase computation demands. Best practice is to add breach locations for the most significant contributors to the expected annual probability of failure; essentially where the levee was expected to have performance concerns. This approach will best yield results consistent with how risk assessors intended. PFMs with a lower probability of occurrence compared to the governing PFMs are not expected to trigger in the modeling and contribute to FFRD products. Levee specific performance information of levee features including earthen embankments and floodwalls can be applied directly. The quantity of breaches will vary but typically will have a breach location for overtopping and at least one breach per feature (earthen embankment, floodwall, etc.). This approach increases the likelihood that FFRD results will be aligned with published risk assessment results.

    When more detailed risk assessment information is not available, a minimum number of breaches should be implemented based on levee length to improve chances of characterizing inundation area from levee breaches.

    Table 1 provides the recommended minimum number of levee breaches per levee length. The breach quantity can be adjusted based on engineering judgment. The actual number of breaches may increase based on the underlying risk assessment data supporting the evaluation. When siting multiple breach locations, breach locations should be spaced somewhat evenly and prioritize another breach location in the upstream increment of the levee system. For example, if a levee is recommended to have two breach locations and the LCL is located in the lower half of the levee, a representative, prior-to-overtopping location should be sought in the upper half of the levee system. Engineering judgment is required (See Section 1.5.2).

    Levee Breach Location¶

    Once the target quantity of breaches per levee system is determined, the breaches can be located. Breach location selection is generally based on current USACE Mapping, Modeling, and Consequences (MMC) Production Center’s Technical Manual for Levees. Locations are identified using the following prioritized considerations:

    1. Breach location at the incipient overtopping location, also known as the LCL, as understood at the time of model setup including using initial without breach model runs. Note that this will likely require hydraulic modeling team input from initial calibration runs and be finalized iteratively.

    1. Known areas of poor expected performance based upon risk assessment results.

    2. Locations of maximum levee height. Do not select areas with infrequent loading or those significantly lower than typical overtopping annual exceedance probability (OT AEP). Refer to loading function, as available. Taller levee sections have more potential susceptibility to various PFMs and would yield significant breach inflows.

    3. Location in the upstream third of the levee system.

    4. High population or structure density areas adjacent to the levee.

    While this list is provided in priority order, combinations of factors may present a stronger case than a rigid hierarchy. For example, a representative breach location adjacent to a population center in the upstream third of the levee may be considered a more critical location than a maximum height levee at the downstream end of the levee. FFRD steering committee guidance stated that FFRD flood hazard data is important for land use planning as it is for current structures; therefore, locating breaches to inform flooding to populated areas is a minor consideration overshadowed by the need to model most likely overtopping and breach locations. Engineering judgment is required. Location selection rationale must be documented.

    Multiple Breach Locations for a Levee System¶

    The SRP curves should be as representative of each unique breach location identified. For example, a single breach location at the LCL will have a simplified assumption that all critical prior-to-overtopping performance is combined and applies at the same location most likely to overtop. For levee systems with multiple breaches assigned, breaches will be assigned performance data that reflects the reason for selecting the location. For example, a levee system that has a LCL and two likely breach locations will have the breach locations represent the system response for those failure modes and the LCL represents the remainder of the performance evaluated during the risk assessment.

    Breach Locations from Risk Assessments¶

    USACE risk assessments help inform levee performance during loading. The assessment may or may not document a likely breach location. Further, establishing breach locations from USACE risk assessment reports often requires manual searching through reports including Qualitative Risk Assessment (QRA), Semi-Qualitative Risk Assessment (SQRA), and Levee Screening Too (LST) 1.0 risk assessments.     Breach Location Selection for Levees with Levee Screening Tool 1.0 Data¶

    Screening-level risk assessment data is available in the LST. LST 1.0 data is available from the “Legacy Screening” round (Figure 1). The levee user is seeking information that will inform a breach location(s) and this can be done by focusing on what scenario drove risk (overtopping or prior to overtopping breach). Reference the annual probability of breach field and the annual exceedance probability (AEP) top field for a comparison.

    If the overtopping likelihood is more likely, the LCL should be selected for single breach levee systems. Information on the LCL may be discussed in the LST print-all found in the Levee Senior Oversight Group (LSOG) meeting folder. Additional supporting documents, including hydraulic and hydrologic memorandums or water surface profile versus crest comparison data may be available. Not all screenings will specifically identify an LCL, in those cases tentatively select a localized low point to be validated by HEC-RAS modeler runs.

    If the prior to overtopping condition is more likely, evaluate which performance mode was assessed as most likely to occur and locate the justification language in the print-all or PowerPoint presentation. Seek references to a specific location that was the primary concern during the risk assessment. Not all screenings will specifically identify a performance weak point, in those cases tentatively select a breach location following general hierarchy guidance.

    Team Communication¶

    Selected levee breach locations for the project are stored in a Levee Breach Locations geospatial file. This file is established during the Scoping and Data Preparation Phase of the project and adjusted as necessary through the subsequent phases of the project. This file documents breach locations, names, key attributes, and rationale for the selection of each location.

    A template for this Levee Breach Locations file is included by Appendix C of the SOP.

    The levee data lead is responsible for creation of this file during the scoping and Data Preparation Phase and will be available to the hydraulic modelers at the start of model development activities during the Calibration Phase. The hydraulic modelers must collaborate with the levee data lead to make necessary adjustments.

    The team may also use a spreadsheet to manage interim communication of breach locations and attributes to gain a consistent understanding of breach elevations, selection criteria, and scenario specific details.

    Consolidation of Existing Risk Assessment Information¶

    The SRP curves developed for FFRD modeling are uniquely developed and dependent on the understanding of the subject levee’s expected performance during flood loading as determined by a risk assessment. The National Levee Database data can identify if a risk assessment was completed (LSAC Rating field) and if a higher-level risk assessment was completed (Has QRA/SQRA (Yes/No) field). The levee data lead identifies all sources of levee performance data used to develop SRP curves.

    System Response Probability Curve Development¶

    An SRP curve represents the conditional probability of breach for a levee location. Figure 2 shows an example of a levee SRP curve plotted in terms of elevation versus probability of breach. The system response curve, a cumulative distribution function, provides a conditional probability of breach for prior-to-overtopping and overtopping events. The SRP curve begins at the origin (0,0) and increases monotonically.

    The relationship of system response and annual probability is important to understand prior to developing SRP curves. Figure 3 provides an illustrative example of the area under an SRP curve and what that equates to in an annualized value. The curve is simplified to show p(f) of 1.0 at flood frequencies (AEP) exceeding the crest. The x-axis is the AEP of a flood loading event and the y-axis is the conditioned probability of levee breach. In the figure, the area under the entire system response curve is the annual probability of inundation (API). The area under the portion of the curve prior to overtopping AEP is referred to as the annual probability of breach prior to overtopping (APFPTOT) and the remaining portion of the curve represents the overtopping contribution to the API.

    {APF\ }_{TOT} = \ {APF}_{PTOT}\  + \ {API}_{OT}

    Figure 2. Example of a Levee System Response Curve: Elevation (feet) versus Probability of Breach. Dashed line indicating the overtopping elevation. The dashed line in the figure indicates the overtopping elevation for the breach location.

    Figure 3. Example System Response Curve with key annualized probability terms represented.

    A few key terms are important to define to build the understanding of how SRP curves are developed:

    PFM: A PFM is a mechanism that once initiated potentially could progress to breach of a levee system or inundation of the leveed area.

    Probability of Failure (p(f) or p(fail)): The probability of levee breach conditioned upon a given water surface elevation, percent levee height, or flood frequency.

    Annual Probability of Failure Prior to Overtopping (APFPTOT): The annual probability of failure (APF) represents the likelihood, in any given year, that a levee will breach considering the full range of potential loads to which it could be subjected (LST Technical Reference Manual). The APFPTOT is a summation of only the likelihood of breach for loading below the levee crest.

    API: The API is the APFPTOT plus the annual probability of overtopping with and without breach.

    OT AEP: The probability that levee overtopping will occur in any given year considering the full range of possible annual floods.

    SRP curve development utilizes risk assessment information and/or USACE levee loading and incident data. The development process is dependent on if the levee has been assessed with a risk assessment or not. Further, if the levee does have a risk assessment, the type of risk assessment informs SRP curve development.

    Tiered Approach to System Response Probability Curve Development¶

    SRP curve development uses existing risk assessment information and/or USACE LLID. The SRP curve development procedure varies depending on the type of performance data available. The following represents the tiers of data that are available to characterize a levee’s expected performance given flood loading (in order of highest fidelity).

    1. QRA

    1. SQRA

    2. LST

    3. Legacy LST (LST 1.0)

    4. No Risk Assessment

    QRA and LST risk assessments produce SRP curves for each PFM evaluated. FFRD SRP curves developed from these sources are combined, distributed, and/or down-scaled appropriately for the levee breach location scheme.

    SRP curves developed from SQRA and LST 1.0 will use the risk assessment output values of APF for each PFM. An SRP curve is iteratively generated using the risk assessment APF as a target and adjusting log-normal shape parameters until the computed APF matches the target APF. FFRD SRPs developed are then combined, distributed, and/or down-scaled appropriately for the levee breach location scheme.

    Levee systems without levee-specific performance data will be assigned a system response of an average levee based on the USACE LLID. FFRD SRPs developed are then combined, distributed, and/or down-scaled appropriately for the levee breach location scheme.

    Intervention¶

    In most modern risk assessment procedures, the ability for to detect and intervene to arrest a breach mode during a flood is already included. For legacy levee performance data, including LST 1.0, an intervention correction is applied using LLID parameters. The LLID data for average levee performance including intervention will be applied for all levees without levee-specific data.

    Multiple Overtopping Locations on the Same System¶

    In consideration of multiple levee breach locations, every breach location assigned will include an overtopping system response. This will model a realistic scenario in which all potential areas of overtopping have a likelihood of breaching from overtopping. USACE subject matter experts (SMEs) concluded this approach creates a situation for the model to perform its own competing risk calculation, minimizing the length effect concern. Theoretically, as the initial overtopping location begins to overtop and possibly breach, the river stage at the other breach locations is less likely to achieve an elevation needed to trigger a breach. Thus, each of the individual breach locations of a particular levee system are competing against each other for the chance to breach. Once one is successful, the probability of another occurring drops so as not to introduce length effect issues of the probability of failure approach.

    Starting with a Quantitative Risk Assessment¶

    From a QRA report, the probability of failure for a given percent loading, flow, or event AEP is typically available directly, but this information needs to be manually collected and then assigned to the breach location setup previously determined. Breach location assignment may involve combination of one or more PFMs and the addition of expected overtopping performance. If loading function information is available at multiple locations and varies significantly, the PFM specific system response should be converted to an AEP versus p(f) format before combining.

    Figure 4 provides an example of input data collected from a USACE QRA report. In this particular example, the x-axis is provided in river flow and will need to be converted to percent levee height or AEP depending on if the system response for this PFM is a standalone breach location or requires combination with other PFMs. In order to convert the axis, the user will collect information from the report that best provides an understanding of the flood loading function and coincident loading reference points on the levee.

    Figure 4. Example System Response Probability Curves for a Single Performance Mode Included in a USACE Quantitative Risk Assessment Report

    Combination of likelihood of breach for multiple PFMs into a single breach location is accomplished with simple addition using the formula below. This calculation assumes the PFMs are mutually exclusive. The combination also assumes that the frequency of flood loading is common at a given percent loading in order to arrive at the intended APF for the system. For example, combination of two SRP curves for different PFMs at a 50 percent loading must mean that the 50 percent loading at each of the locations corresponds to approximately the same frequency of flood loading (i.e., 50 percent loading at both locations is equivalent to a 1 in 50 AEP).

    {P(fail)}_{Total\ } = \ {\sum p(fail)}_{i}

    Starting with a Semi-Qualitative Risk Assessment¶

    In most cases, a semi-quantitative risk assessment will not include SRP curves for each PFM. The output users will be looking for from SQRA will be loading functions and the evaluated PFMs and their respective elicited APF with intervention. The general process for developing SRP curves from this data is to set the APF as a target and develop a SRP curve where the area under the curve match that APF target. The shape of the curve is influenced by log normal shape parameter (sigma) that are from USACE LLID. USACE maintains a spreadsheet for this computation. The detailed steps are:

    1. Evaluate hydraulics and hydrologic data in the report. Locate and enter one or more flood loading functions required to represent the flood loading for the PFMs. Input data needed: Stage and AEP for each loading curve. Enter the incipient overtopping point to document the cutoff point for APF calculation.

    1. Locate and enter the APF with intervention for each elicited PFM as the target APF for the calculated SRP for each PFM. All credible breach modes from the SQRA report will be used to ensure that the total APF for the system is considered. Assign each PFM to a standard PFM type to associate the LLID shape parameters.

    2. Perform log-linear interpolation of each percent loading function to a standardized set of nodes to normalize all PFM entries to a common set of AEP range so SRPs can eventually be combined.

    3. Set an iterative computation of SRP at each AEP node by varying shape parameter (σ), sigma ln, until the computer APF of the created SRP curve matches the target APF from the SQRA report.

    4. Repeat for all credible PFMs.

    5. Combine all or portions of the calculated SRP to match the desired distribution of PFMs at the breach locations. A system represented by only a single breach location will sum all calculated PFM SRP curves. A levee system represented by multiple will use specific PFM SRP curves directly and sum the remainder at the LCL, for example.

    Figure 5 shows the calculation spreadsheet setup with orange cells indicating entry. Fail mode is selected from a set dropdown list. User selects the loading function that applies to this PFM. User enters the elicited PFM APF with intervention from the SQRA report. Finally, an iterative approach is used to change sigma ln until the computed APF matches the target APF. The prior to overtopping SRP curve is now created for this PFM. The expansion of percent levee height is a log-linear interpolation of the input flood loading curve. The calculation of SRP is done as a cumulative density function with a log-normal distribution for the given percent levee height ordinate with mean (μ) and standard deviation (σ). Finally, the computation of APF is computed as the area under the SRP curve using a Riemann sum trapezoidal approximation:

    APF = \sum_{}^{}{\left( x_{i} - x_{i + 1} \right) \times \left( y_{i} + y_{i + 1} \right)} \div 2

    Starting with a Screening Level Risk Assessment¶

    Screening level risk assessment data is available in the LST. Once a multi-disciplinary team completes a screening, the LST maintains a database information on levee performance (PFMs), representative breach locations assigned to those PFMs, and representative loading functions assigned to those PFMs. The LST uses a standard set of PFMs that are assessed depending on the features of the levee (e.g., earthen embankment, floodwall, and/or closure structure). Overtopping with breach for the incipient overtopping location is also assessed.

    Within the LST, a large database export can be accomplished with administrator privileges in the Reports tab on the main banner. SRP curves can be extracted on a levee-by-levee basis within the Computations tab of a levee segment.     Starting with a Screening Level Risk Assessment (LST 1.0)¶

    The most prevalent levee performance on levees in the United States comes from the first version of the LST (LST 1.0) with a screening level risk assessment completed for the entire USACE portfolio.

    The input data needed to develop SRP curves from this data set is the LST 1.0 database export file from October 2023. The file contains levee toe and crest loading frequency (AEP) and performance indexes for each PFM. The user should understand that it is typical that only a single generalized loading curve representing the most frequent loading toe AEP and the incipient overtopping frequency are available. A segment may have a separate loading function for embankments, floodwalls, and/or closure structures if those features are present. Further, the levee performance was based on a relative risk Bayesian updating approach. Conversions of the performance data are included to correct the performance estimates to expected magnitudes and shape parameters from the USACE Levee Loading and Incident data as well as corrections to account for intervention. Screenings were performed on levees at the segment level. One or more segments may comprise a levee system. The following procedures are for a combined SRP curve at the segment level. Single breach setups for a multi-segment system will combine the curves as previously described in the p(fail) versus AEP format. Multiple breaches assigned to a single-segment will require splitting relevant PFMs to each breach location as appropriate.

    Given that all input data is being pulled from a single dataset, the process is mostly automated once computational setup is achieved. USACE maintains a spreadsheet for this computation. The detailed steps are:

    1. Enter the NLD segment ID of the subject levee segment in the Loading tab

    1. Extract the LST flood loading data. Toe and crest AEP estimates are available for the segment and additional entries for floodwall loading and closure structure loading, if applicable.

    2. Select Yes for every applicable PFM. The LST used a standardized set of PFMs applicable to a feature type (embankment, floodwall, and closure structure).

    3. Assign the correct loading function to each PFM.

    4. On the Combined Curve tab, the target APF will be pulled from the LST 1.0 data. The intended LST 1.0 field is the performance index. There is a performance index associated with each evaluated PFM from the standardized suite of LST PFMs.

    5. Iteratively adjust the sigma ln shape parameter until the calculated APF for the SRP curve you are creating matches the target APF.

    6. Repeat for all credible PFMs.

    7. The spreadsheet will perform the computation to account for intervention (Figure 9).

    8. Combine all or portions of the calculated SRP (adjusted for intervention) to match the desired distribution of PFMs at the breach locations. A system represented by only a single breach location will sum all calculated PFM SRP curves. A levee system represented by multiple will use specific PFM SRP curves directly and sum the remainder at LCL, for example.

    System Response Probability Curve Conversion¶

    During SRP curve creation, the format most likely will need to change axes formats from initial AEP versus p(f). In addition, the required a standard number of ordinates on the FFRD curves may require an expansion of contraction of a calculated curve. Finally, the FFRD HEC-RAS model requires an elevation versus p(fail) format using the elevations at the breach location (North American Vertical Datum 1988 datum). As Figure 10 illustrates, the AEP axis is then converted to a percent height and finally to a usable format for FFRD to the breach location selected in an elevation versus p(f) format.

    Figure 10. System Response Probability Curve Format Conversion Process

    1. The SRP curve generation process requires establishment of a related percent levee loading relationship to the flood loading. This step is to revert the AEP axis back to a percent levee height scale. This is available in all USACE prepared computational spreadsheets. The user must ensure that the scale represents the percent loading for the intended breach location.

    1. The following step is to convert to the standardized number of ordinates. Currently, the SRP curve generation spreadsheets are setup to generate an SRP curve with 10,000 ordinates to conform to FEMA insurance pricing requirement; however, depending on the development method the raw curve may have a different number.     System Response Allocation, Combination, and Formatting¶

    SRPs representing performance for a specific PFM or levee segment are then combined to allocate system response to the defined breach location scheme. If the breach scheme identifies the need for a prior-to-overtopping breach location and the risk assessment source has identified a critical flaw at this specific location, the full system response for that breach mode is applied. If no critical location is identified, the combined performance modes evaluated in the risk assessment are down-scaled and applied evenly to all breach locations. The combination computation applies De-Morgan’s rule for statistically independent and not mutually exclusive PFMs when the risk assessment data is unadjusted (Table 8).

    $Probability\ of\ Failure = 1 - \left( 1 - p(fail) \right)^{N}$

    Table 8. Summary of Adjusted and Unadjusted System Response by Risk Assessment Type

    Risk Assessment Type Adjusted or Unadjusted Combination Method
    QRA Unadjusted De-Morgan’s Rule
    SQRA Unadjusted De-Morgan’s Rule
    LST (Current) Adjusted Summation
    LST (v1.0) Adjusted Summation

    When levee SRPs need to be distributed to multiple identified breach locations, an adjustment to De-Morgan’s rule is applied to the conditional probability of failure for the system (P(fail)) to statistically down-scale the system response the number of breach locations (NLocations).

    Probability\ of\ Failure = 1 - \left( 1 - P(fail) \right)^{{1/N}_{Locations}}

    Once the SRP is developed for the breach location scheme, the data is converted to the correct stage range from levee toe elevation to crest elevation. The overtopping response is added using De-Morgan’s rule to the prior-to-overtopping system response, assuming that prior-to-overtopping at 100 percent of the levee height remains constant. Finally, if the probability of breach has not reached 1.0, the SRP curve is completed by interpolating from the overtopping response.

    Combining Overtopping performance to Prior to Overtopping¶

    Risk assessments typically document overtopping differently and include a direct SRP curve that expresses probability of failure as a function of depth over the crest of the levee at that location. Therefore, this portion of the SRP curve will be added after the SRP curve is in the elevation versus p(f) format.

    The elicited overtopping performance typically has a maximum range (i.e., only elicited to 2 feet over the levee crest height). For the purposes of FFRD, the overtopping performance will be extrapolated to 99.9 percent chance of breach and then given a straight line 100 feet above the levee crest elevation to ensure that the sampled elevation range exceeds the potential FFRD hydraulic model elevation.

    The final step when combining prior to overtopping and overtopping portions of the SRP is to consider that prior to overtopping PFMs have a non-zero chance of breach, even during overtopping. For example, a levee may still breach from internal erosion during overtopping so it will be combined with likelihood of overtopping with breach; however, tailwater conditions as the leveed area fills ultimately cause the internal erosion SRP to plateau. To account for this, the prior to overtopping probability of failure is held as a constant during the addition of the overtopping performance probability of failure. In reality, this is not a constant, but this approach is used as a simplification to not ignore this contribution.

    Format for Levee System Response Probability Curve Delivery¶

    The SRP curves are compiled, quality control reviewed, and then uploaded to Model Library as a single .CSV file following the format as Figure 11 shows. The file includes the 2D connection name in HEC-RAS model to link the system response trigger elevation sampled from the system response curve to the water surface modeled in HEC-RAS. The remaining common data fields are data identifiers including the National Levee Data system and segment ID numbers, and the basis for the system response curve (i.e., risk assessment source, potential failure mode, or default performance). The system response includes prior to overtopping and overtopping performance.

    The provided SRPs formatted as .CSV files will be used as an input for the cloud-compute actions that sample breach trigger elevations for the stochastic simulations.

    A template for use in creation of the required SRP format for delivery in included in Appendix C, named system-response. Note that no commas are allowed in the header information fields.

    The levee SRP .CSV file should be initially delivered to the project directory in the ‘[basin-name]>calibration>system-response’ path.

    Review Evaluation of Breach Results¶

    A single realization of stochastic events is simulated by cloud compute during the conformance phase. Breach results from the conformance phase must be reviewed by the dam data lead to ensure that breach trigger elevations are being sampled appropriately and the number of breach events that occur at each breach location falls within the expected range. Similarly, the multi-realization results from the projection phase must be reviewed and evaluated.
