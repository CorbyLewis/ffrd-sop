### *Calibration Phase*

*Model calibration is performed with the intent of developing event-specific modeling results which can be used by other models in the modeling sequence. Additionally, acceptable model parameters that can be used within probabilistic/hypothetical simulations are required and obtained through historical event simulations. Due to the nature of the FFRD modeling environment, results from one model can serve as inputs to another model. This is the case for hydrologic model results which serve as the primary inputs for the reservoir and hydraulic models. Due to these dependencies, there may be cases where iteration is required between models to achieve acceptable model calibration results. In these cases, coordination between the hydrologic modeler, hydraulic modeler, reservoir operations modeler, and/or technical team lead may be required to achieve successful calibration across the suite of FFRD models.*

#### *Hydrologic Model Calibration*

*The performance of the hydrologic model must be evaluated by comparing computed results against observed results at each primary, secondary, and tertiary location. Model parameters should be altered in consistent and tractable ways in order to minimize the differences between computed and observed hydrograph shape, peak flow rate, and discharge volume.*

*Additionally, if the watershed is hydrologically impacted by snowmelt, model parameters should be altered in consistent and tractable ways in order to minimize the differences between computed and observed snow water equivalent (SWE). Generally speaking, matching the shape and magnitude of observed SWE will result in acceptable streamflow calibration. However, it is possible to closely match observed SWE accumulation / melt and still under- or over-predict runoff rates. Therefore, it is preferred to more closely match runoff rates in lieu of SWE when the two observed data sets result in different parameters to achieve acceptable calibration.*

##### *Hydrologic Model Calibration Targets*

*Targets for model calibration are discussed within the main Standard Operating Procedure (SOP) document. This includes statistical metrics required for primary, secondary, and tertiary gages. These statistical metrics are applied to event-based stage and flow hydrographs.*

*When adequate observed data is available (i.e., at a minimum, daily average streamflow), summary statistics must be used to quantify model performance at all primary, secondary, and tertiary locations (Moriasi, et al., 2007). Required statistical metrics include Nash-Sutcliffe Efficiency (NSE), Ratio of the Root Mean Square Error to the Standard Deviation (RSR), Percent Bias (PBIAS), and Modified Kling-Gupta Efficiency (MKGE). In addition to summary statistics, model calibration includes a graphical comparison of the modeled stage and/or flow with observed data at all gage locations with available data. These comparisons are included in the calibration process and model adjustments are made as needed.*

####### Loss Parameter Considerations

*Deficit and Constant loss parameters should be modified to match the initiation of runoff, observed runoff volume, and peak of the observed discharge hydrograph. Loss parameters should be modified in a regionally consistent and tractable manner. For each subbasin, ensure that the Maximum Deficit and/or Percent Impervious parameter do not vary from one historic event to the other. If they do vary, an adequate physical justification must be provided. Loss parameter modifications should be made in conjunction with baseflow and transform parameters. The following resource provides helpful calibration guidance: [Applying the Deficit and Constant Loss Method](https://www.hec.usace.army.mil/confluence/hmsdocs/hmsguides/applying-loss-methods-in-hec-hms/applying-the-deficit-and-constant-loss-method).*

####### Transform Parameter Considerations

*Modified Clark transform parameters should be modified to match the observed hydrograph shape and peak discharge magnitude and timing. Transform parameters should be modified in a regionally consistent and tractable manner. Dimensionless coefficients of the transform parameters (e.g., R / \[Tc + R\]) should be used for regional comparisons. Care should be taken so that the computed direct runoff hydrograph peak occurs prior to the baseflow hydrograph for each subbasin. Also, Tc must be greater than the computational time step while R must be greater than 0.5 \* computational time step for all subbasins. Transform parameter modifications should be made in conjunction with the loss and baseflow method parameters. The following resource provides helpful calibration guidance: [Time of Concentration and Storage Coefficient Impacts to the Hydrograph](https://www.hec.usace.army.mil/confluence/hmsdocs/hmsguides/applying-transform-methods-in-hec-hms/estimating-clark-unit-hydrograph-parameters/time-of-concentration-and-storage-coefficient-impacts-to-the-hydrograph).*

####### Baseflow Parameter Consideration

*Linear Reservoir baseflow parameters should be modified to match the shape of the receding limb of the observed discharge hydrograph as well as the observed runoff volume. Baseflow parameters should be modified in a regionally consistent and tractable manner.*

*Within FFRD HEC-HMS models, the linear reservoir baseflow method is typically defined using two groundwater (GW) layers. However, a third layer may be required depending upon physical considerations of the modeling domain in question. The GW 1 layer must be conceptualized as conveying interflow or quick-responding subsurface water while GW 2 layer must be conceptualized as conveying slow responding baseflow. As such, GW 1 parameters must never be used that result in GW 1 outflow preceding direct runoff provided by the transform method. Similarly, GW 2 parameters must never be used that result in GW 2 outflow preceding outflow from GW 1.*

*Baseflow parameters should be modified in conjunction with the loss and transform method parameters. The following resource provides helpful calibration guidance: [Applying the Linear Reservoir Baseflow Method](https://www.hec.usace.army.mil/confluence/spaces/HMSGUIDES/pages/41419107/Applying+the+Linear+Reservoir+Baseflow+Method).*

####### Reach Routing Parameter Consideration

*Reach routing parameters should be modified to match the observed hydrograph translation and attenuation. Reach routing parameters should be modified in a regionally consistent and tractable manner. Care must be taken to produce outflow hydrographs for all routing reaches that are devoid of oscillations. When using the Modified Puls method, storage-outflow relationships must be extended to avoid errors when simulating events that are significantly larger in magnitude than the historic events chosen for calibration. The following resources provide helpful calibration guidance:*

- [*Applying the Muskingum Routing Method*](https://www.hec.usace.army.mil/confluence/hmsdocs/hmsguides/applying-reach-routing-methods-within-hec-hms/applying-the-muskingum-routing-method)

- [*Applying the Muskingum-Cunge Routing Method*](https://www.hec.usace.army.mil/confluence/hmsdocs/hmsguides/applying-reach-routing-methods-within-hec-hms/applying-the-muskingum-cunge-routing-method)

- [*Using an HEC-RAS Model to Create Storage-Discharge Curves for Mod-Puls Reaches in HEC-HMS*](https://www.hec.usace.army.mil/confluence/hmsdocs/hmsguides/applying-reach-routing-methods-within-hec-hms/using-an-hec-ras-model-to-create-storage-discharge-curves-for-mod-puls-reaches-in-hec-hms)

####### Snow Parameter Considerations

*Snow parameters should be modified to match the observed accumulation and melt of SWE in a regionally consistent and tractable manner. Subbasin average SWE time series should be used to calibrate snow parameters. These time series are extracted from observed snow grids as discussed within Job Aid 5 – Gathering and Summarizing Observed Gridded Data and Job Aid 6 – Gathering Observed Gage Data. Initial snow state variables (e.g., initial SWE) should be set using the same observed snow grids, when available. Care should be taken so that the wet melt rate is always greater than or equal to the dry melt rate for antecedent condition for each subbasin. If a groundmelt rate greater than 0 is used for any subbasin, an adequate physical justification must be provided. The following resource provides helpful calibration guidance: [Calibrating Gridded Temperature Index Parameters](https://www.hec.usace.army.mil/confluence/hmsdocs/hmsguides/modeling-snowmelt-in-hec-hms/calibrating-gridded-snowmelt-upper-truckee-river-california/option-1-gridded-temperature-index-upper-truckee-river).*

Simpler reservoir routing methods and parameterizations should be favored to minimize computational burdens. Observed data can be applied to help the model target a storage pattern when using the Rule-Based Operations method or to build an outflow discharge curve when using the Outflow Curve method. ###### *Ungaged Basins*

*Observed data may be limited or unavailable for some areas or some event time frames, this constraint makes calibration more difficult as modeled timeseries data cannot be compared with observed timeseries. In these cases, other data may be valuable for calibrating or validating hydraulic model results. Specifically high-water marks, USGS field measurements, satellite or aerial flood imagery, and other data sources can be valuable sources of observed data in ungaged basins. Note that hydraulic models should <u>not</u> be calibrated to flow regression equations, as the frequency of a modeled storm may not translate well to the flow frequency of a specific stream of interest. These regression equations can be compared to model results but are not a valuable calibration data source.*

*The USGS Flood Event Viewer (available at <https://stn.wim.usgs.gov/FEV/>) is one central source of flood event data, including high-water marks and rapid-deployment stream gages. Many other local databases of high-water marks may exist as well.*

*Satellite imagery may be available and provides a spatial snapshot in time from a flood event. Some sources for satellite imagery include the USGS Earth Resources Observation and Science Center (<https://www.usgs.gov/centers/eros>) and USGS Earth Explorer (<https://earthexplorer.usgs.gov/>). Additional discussion is included in Chapter 3 of the main SOP document.*

###### *Use of Reasonable and Commonly Accepted Model Parameters*

*If calibration is difficult or impossible to perform using reasonable and commonly accepted model parameters, this may indicate a larger structural problem with the model. It may be possible to force a model to match observed data for a single event with parameters that are outside of the range that would be considered normal for that stream, but the model may not work well on a range of events. It is recommended to stay within a realistic range for model parameters.*

*For example, if extreme values of channel roughness are required to reproduce observed data, the model may likely be missing some aspect which is impacting model results. In this case, reexamine 2D cell meshing and other factors discussed throughout this document for potential issues. Also consider factors such as gage data reliability, datum conversions, backwater influence, hydraulic* *structure influence, and/or changed hydraulic conditions which may not be captured by modeling. For extreme floods, it is possible that floodwater flow may flank around the gage location and not be accurately reported in gage records. Similarly, within the hydraulic model, it is possible that reference lines used to report modeled flow and rating curves may not extend fully across the floodplain and may therefore not be capturing all the flow which passes the gage location.*

*For some model parameters, commonly accepted values may not exist and are actively being researched. This is the case for overland Manning’s roughness coefficients, which are used to define upland areas that experience very shallow flow induced by rainfall. Unlike stream channel and floodplain roughness coefficients which are well understood and documented, there are not established standard values for Manning’s coefficients for very shallow precipitation-driven overland flow. This presents a problem for FFRD hydraulic model development, where overland roughness coefficients are required to describe the rainfall-runoff response across the watershed. Engineering judgement is required for selecting these values and guidance is being developed as the FFRD modeling process matures.*

##### *Techniques for Hydraulic Model* *Calibration*

*There are a variety of approaches which can be taken when calibrating a hydraulic model, refer to other resources such as the HEC-RAS user manual for additional information on this topic. The approaches described below are listed in the recommended order they should be performed, however some calibration efforts may require iteration between steps to achieve a satisfactory calibration.*

###### *Stage-Discharge Rating Curves as Calibration Aids*

*One of the primary tools useful for performing hydraulic model calibration is graphical examination of the stage-discharge rating curve. An example of such a rating curve is shown in [Figure 3](#_Ref191565005) and compares model results versus published USGS data and measurements. Focusing on a rating curve during calibration removes the hydrograph timing variable from consideration and can improve calibration efficiency.*

*The lower portion of the rating curve defines the flow and stage characteristics of the stream channel while flows are contained within the banks of the river. Using [Figure 3](#_Ref191565005) as an example, the in-channel portion of the rating curve is below an elevation of approximately 1,533-ft and a flowrate of approximately 2,000-cfs. The upper portion of the rating curve describes flows which are out of the river channel’s banks and can be heavily influenced by floodplain characteristics. For many riverine systems, the elevation-discharge rating curve will exhibit an inflection point near the top-of-bank elevation where flows exceed channel capacity and begin to activate the floodplain. Terrain DEMs and existing model results can also be used to identify bankfull elevations and channel capacities.*

*When calibrating a hydraulic model, care should first be taken to calibrate model results to flows contained within the river channel. Assuming global model parameters are reasonable (such as computation timestep), focusing on the lower in-channel portion of the rating curve constrains the model parameters available for calibration. In general, reasonable in-channel model parameters can be selected and set first, before moving forward to calibrate the model to the higher out-of-bank portion of the rating curve. This approach reduces the variables used to calibrate the high-flow portion of the rating curve, increasing efficiency and reducing iteration.*

###### *2D Cell Alignment and Spacing*

*It is generally a best practice for 2D cell faces to be aligned with the direction of flow as best as is practicable. This can be done through a combination of breaklines, refinement regions, and manual manipulation of 2D cell points. 2D cell face alignment with flow has benefits of computational efficiency and stability, and it can also impact modeled water surface elevations and velocities. 2D cell faces should be aligned to flow prior to detailed calibration efforts. This alignment is also an opportunity to ensure terrain features near the channel are appropriately captured by the 2D cell faces. Features such as levees (both natural floodplain deposits and human-made levees), as well as other distinct features along banklines can be captured using breaklines and refinement region boundaries. These is also a convenient time to adjust cell spacing within the stream channel. Due to the large number of streams present in FFRD hydraulic modeling units, it may not be practical to align all 2D cells to stream flow. Larger streams and those near developed areas should be prioritized for cell alignment, this includes streams identified as key and secondary streams.*

###### *Estimation of Channel Bathymetry*

*If high-quality channel bathymetry is available for a given reach, this simplifies calibration efforts associated with that reach as bathymetry does not need to be estimated. If bathymetry is not available, the terrain DEM likely represents the river channel with the water surface filled-in to what the water level was when LiDAR data was collected. For small streams where flow depths are usually small, there may not be a need to approximate terrain bathymetry as it is already well enough defined by the terrain DEM. However for larger rivers, when the channel bathymetry is not reflected in the terrain, channel conveyance can be significantly underestimated and there is a need to approximate the bathymetry. Bathymetry estimation is one of several parameters available for calibrating modeled results to a published rating curve.*

*When channel bathymetry needs to be approximated, the HEC-RAS terrain modifications can be used. These linear channel modifications require several parameters be estimated, namely: a profile of channel invert elevations, channel shape, and channel width. One way to approximate the channel invert elevation profile is to export a terrain profile from a stream centerline drawn on the terrain DEM which does not have bathymetry estimated. This stream profile exported from the LiDAR based DEM can then be lowered by a specified height to estimate the depth of water under the LiDAR based DEM stream surface. To inform this depth of water, the lowest portion of the elevation-discharge rating curve can be used. As an example, in [Figure 3](#_Ref191565005), the channel invert is approximately elevation 1,528-ft, as this is the elevation corresponding to zero discharge. If the terrain DEM indicates a channel invert of 1,531-ft with no bathymetry estimated, the flow depth required to reach the channel invert indicated by the rating curve is 3-ft. Therefore 3-ft can be subtracted from the stream elevation profile which was based on the LiDAR terrain. While this approach of subtracting a constant height from the LiDAR elevation profile is simplistic, it can be an appropriate technique in the absence of more detailed bathymetric information.*

*For many streams, the channel’s cross-sectional shape can be approximated as a trapezoid. The bottom width and side slopes of this trapezoid can be estimated from examination of LiDAR based terrain and aerial imagery. An example of a channel modification to estimate stream bathymetry is shown in Figure 3.3. In this example, channel bottom width and side slopes were estimated from a LiDAR DEM, and the channel station-elevation profile was adjusted to achieve a channel invert as indicated by a published USGS rating curve. Note the offset (3-ft in this example) present between the LiDAR terrain DEM and the estimated bathymetric stream profile.**\*

###### *Selection of Channel Roughness Values*

*Once channel bathymetry is estimated and its parameters (e.g. cross-section shape, width, and depth) are constrained by reasonable values (determined from the LiDAR DEM/aerial imagery), channel roughness can be estimated. This Manning’s n roughness value is a calibration parameter and should be selected to match modeled water surface and flow results to a published rating curve. Specifically the roughness value used for the stream channel should be used to match the lower portion of the rating curve where flows are contained in-channel. Manning’s n calibration regions can be used in HEC-RAS to define specific n values in specific areas. It is often desirable to buffer a stream centerline to the width of the stream channel such that areas wetted during low flows have their own uniquely defined roughness coefficients.*

*Through modification of bathymetry estimates and channel roughness, the in-channel portion of the rating curve should be made to match well to published rating curve data. If calibration to the in-channel portion of published rating curves cannot be achieved using commonly accepted roughness values, this may indicate a larger structural problem with the model. Reexamine 2d cell meshing and stream bathymetry for potential issues. Also consider factors such as gage data reliability, backwater influence, hydraulic structure influence, boundary conditions, and/or changed hydraulic conditions which may not be captured by modeling.*

*Note on Spatially Varied Manning’s Roughness*

*For FFRD models where 2D cell sizes may be large, enabling the Spatially Varied Manning’s n on Faces option for 2D areas is recommended and can improve the fidelity of hydraulic roughness on long cell faces. When enabled this option varies the Manning’s roughness coefficient as a function of elevation depending on the relationship between a 2D cell face and spatially varied Manning’s n layer. This option is briefly described in the HEC-RAS user manual here: <https://www.hec.usace.army.mil/confluence/rasdocs/r2dum/latest/development-of-a-2d-or-combined-1d-2d-model/creating-hydraulic-property-tables-for-2d-flow-areas>*

###### *Defining Floodplain Geometry with Breaklines*

*Once the in-channel portion of the rating curve matches well to observed data, focus can be shifted towards representing the floodplain. Capturing terrain features in the floodplain, such as embankments, berms, ridges, and other impediments to flow is of critical importance to accurately representing flood behavior. This is especially true for FFRD RAS models, which may use large cell sizes that can easily misrepresent terrain features if proper breaklining is not performed. There are known limitations of the current RAS 2D meshing system which result in 2D cell faces not snapping properly to defined breaklines. Modelers must take care to ensure cell faces are snapped and that breaklines are properly enforced. Conflicts often arise where multiple breaklines intersect one another, these areas require special attention and manual manipulation of 2D cell points from modelers to ensure that high ground is captured by cell faces.*

###### *Selection of* *Floodplain Roughness Values*

*Once floodplain terrain features are represented by the 2D computational mesh, final calibration to the rating curve can be performed through modification of floodplain roughness coefficients. This effort is focused on the upper portion of the elevation-discharge rating curve where elevations exceed bank-full, as the in-channel portion of the rating curve should be established by previous steps discussed above. Additional information on selection of these roughness values can be found in the* *main SOP document, section 8.9.6 – Roughness Values. Like the in-channel regions of the model, RAS calibration regions can be used to define specific areas where Manning’s n values are modified. Also see the note above regarding the use of spatially varied Manning’s roughness, which can be useful in calibrating floodplain model results.*

*If calibration to the floodplain portion of published rating curves cannot be achieved using commonly accepted roughness values, this may indicate a larger structural problem with the model. Refer to section [3.2.2](#use-of-reasonable-and-commonly-accepted-model-parameters-1) for discussion of reasonable and commonly accepted model parameters.*

###### *Flow Hydrograph Timing and Magnitude*

*Once the hydraulic model is sufficiently reproducing the published elevation-discharge rating curve, flow hydrographs can be examined for proper timing and magnitude. These variables can be influenced by many factors, including: upstream roughness value selection (in the stream channel, floodplain, and upland areas), upstream mesh resolution, refinement, breaklining, 2D equation set selection, and to a lesser degree, hydraulic property table tolerance and other variables. Hydrograph timing and magnitude are also heavily influenced by HEC-HMS hydrologic model outputs, namely gridded excess precipitation and baseflow hydrographs. The interaction of all these variables is complex and engineering experience and judgement are required to achieve satisfactory calibration. To achieve successful hydraulic calibration, it may be necessary in some instances to modify hydrologic model parameters which serve as the primary inputs to the hydraulic model. This* *should be discussed with the technical team lead and hydrologic modeler.*

### *Conformance Phase*

*Models are calibrated to historic events in isolation during the calibration phase of the project. During the conformance phase of the project, the full system of models are linked in such a way that an adjustment to the parameters of one model can affect the outputs from a separate model. For this reason, coordination across modeling teams in necessary to optimize the performance and accuracy of the full model system rather than any single model.*
