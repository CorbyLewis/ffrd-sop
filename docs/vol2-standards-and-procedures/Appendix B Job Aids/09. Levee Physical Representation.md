# Job Aid 9—Levee Representation

This job aid provides guidance for representation of physical levee structures within FFRD hydraulic models.

This job aid also discusses considerations related to levee breaches. The levee data lead also provides draft breach locations prior to the start of the Calibration Phase. Since HEC-RAS only allows a single breach per storage area/2D connection, the breach locations will impact the segmentation of levees that have more than one breach scoped.

## Levee Alignment and Profile Data

The following subsections discuss levee alignment and profile data in the FFRD program.

### National Levee Data Refinement Effort

As described in on other SOP materials, the National Levee Database (NLD) data is the official source for FFRD levee modeling. The NLD data for the study basin will undergo a data refinement effort prior to or as a part of the FFRD Scoping and Data Preparation phase, to ensure the dataset is adequate for direct use in FFRD modeling. This NLD data refinement effort is described by reference documents stored directly in the NLD.

### Sources of Levee Data

As stated previously, the NLD provides a near comprehensive list of levee structures to be included in the hydraulic model and, in many cases, the NLD will be the best available data source. However, there are cases where NLD data may be missing, incomplete, outdated, or otherwise incorrect. Levees are unique and dynamic structures which can change or be modified overtime. Levee raises, setbacks, erosion, or other modifications are examples where the NLD may not reflect the best understanding of the levee structure. For this reason, it is important for the hydraulic modeler to verify accuracy of levee data. This verification can be done through discussion with the local USACE district, through comparison with historic and current aerial imagery, and through comparison with other available data. Other potential sources of for levee data include risk assessments, district modeling and surveys, as-built drawings, and digital elevation models (DEMs) of the terrain, as well as other sources. If a discrepancy is found between NLD data and other sources, modeler judgement and discussion with the technical team lead may be needed to determine the best source of levee data.

One data source that is widely available for comparison with NLD data is a high-resolution DEM of the terrain. This terrain data can be used for verification of NLD crest profile data, however it may not accurately depict specific portions of the levee, such as floodwall sections or closures. If issues are found with the NLD data and a decision is made to use terrain elevations to define the structure crest, it may be necessary to modify the NLD structure centerline to correspond with the high ground indicated by the terrain. Often the NLD centerline will not perfectly align with the terrain’s high ground and maintaining the NLD centerline while recutting the levee profile’s crest data from the terrain can underestimate the true levee crest elevation.

### Communication with Levee Data Lead

If NLD data is not adequate and the modeling team finds an alternative improved data source, it shall be annotated in the Calibration Phase report in the levee system-by-system table. It will also be communicated with the levee data lead for possible NLD data refinement.

### Direct Use of National Levee Data Base Profile and Alignment

Levee data should be adopted from the best available data source, which in many cases will be the NLD. Levee centerline data is available as shapefiles through the NLD and this alignment should be adopted unless better alignment data is available. This NLD alignment may be drawn upstream-to-downstream or downstream-to-upstream. Unless there is a compelling reason to reverse this directionality, the direction of the NLD structure should be maintained. If structure direction is reversed, this and its rationale should be documented in both the hydraulic model report and described in the HEC-RAS Connection Data Editor description box.

The levee centerline data is provided by NLD for the full levee system. This centerline shapefile can be split at levee segments so that each segment is represented by its own connection within the hydraulic model. It is generally best practice to create a small gap between storage area/2D connections which would otherwise touch. This small gap (on the order of 1-foot wide) improves 2D face meshing and can improve stability in some circumstances. When these gaps are added between structure segments, it may also be necessary to add a corresponding gap to the structure crest profile data, as discussed below. An additional technique, which can improve 2D cell meshing, is to underlay a 2D area breakline beneath the storage area/2D connection structures representing a levee system. This breakline can particularly improve meshing in instances where it spans the small gaps that can be placed between levee segments.

Note that depending on the direction the levee structure is drawn, and the side of the river the levee sits upon, the flow over the structure may be computed as either positive or negative. In addition to levee centerline data, levee crest profile data is also available for many levees from the NLD. This data is accessed through the levee system’s Profile tab and can be downloaded as a comma separated value (CSV) file. These crest profile .csv files are typically formatted as x-coordinate, y-coordinate, distance (also known as stationing or m-value), and elevation (z-value). The distance (station, m) and elevation (z) values are of interest for defining the levee crest profile in HEC-RAS. By default, the stationing provided in the NLD .csv file should correspond to the direction of the centerline shapefile discussed previously. Once a connection structure is created in the HEC-RAS model, tabular XY coordinates and station-elevation data can be entered in the HEC-RAS weir/embankment data editor. An appropriate weir crest width and discharge coefficient should be entered in this editor, refer to the HEC-RAS User Manual for additional guidance on selecting weir discharge coefficients. If a structure’s centerline data was modified (e.g., reversed or split), a corresponding modification should be made to the profile crest station-elevation data.

### Levee Model Configuration Details

Levee data should be adopted from the best available data source, which in many cases will be the NLD. Levees should typically be modeled as internal storage area/2D connections. Refer to the Job Aid 06, *Standard Naming Conventions,* for guidance regarding naming convention of levee structures. The HEC-RAS connection data editor includes a text description box, which should be utilized by the modeler to describe the source of levee data and any assumptions made regarding the levee structure. This text description should include the source of the levee centerline alignment, NLD ID number, NLD system and segment names, the source of crest profile data, and any other pertinent data or assumptions.

Levee structures should typically be incorporated into the HEC-RAS model at a segment-by-segment basis, this is especially true of long levee systems. Shorter hydraulic connections are more conducive to evaluation and troubleshooting of hydraulic model results compared with long connections. HEC-RAS is limited to a single breach location per connection, and therefore shorter segment-by-segment connections allow for potential breaching of each segment and multiple breaches in a multi-segment levee system.

#### Overflow Computation Method

HEC-RAS prompts the hydraulic modeler to select the *overflow computational method* to calculate discharge over the structure, with the options of either the *Normal 2D Equation Domain* or *Use Weir Equation*.

2D equations more accurately calculate overflow for levees that do not have significant height above the surrounding terrain, are subject to full submergence during floods, and will generally be more stable than the weir equations but may overestimate overtopping discharge when there is a significant head difference over the levee structure. Weir equations better represent tall levees where the levee is expected to act as a non-submerged weir during flood conditions.

***Weir equations should be used for FFRD models***. 2D equations are not recommended for FFRD models because they are not fully compatible with the simplified physical breach method.

#### Reference Points and Levees

Reference points are required near levees to ensure adequate water surface elevation information is written out by the model in the vicinity of levees. The requirements for these saved output locations including reference points near levees are described in Section 8.3 of SOP Volume II. There are also specific naming convention requirements for these reference points defined within Job Aid 06, Section 1.6.

#### High Ground

In some cases, the levee centerline provided by the NLD may not capture high ground that is not technically part of the levee system but effectively acts as part of the levee structure. [Figure 2](#_Ref191461837) displays an example where the levee centerline, shown in pink, does not capture high ground created by a bridge abutment. Levee systems should be inspected for instances such as these, and if there is adjacent high ground which acts as a water impounding surface, it should be included in the hydraulic model as part of the levee structure or as a separate breakline.

#### Closures

Many levee systems include closure structures which are not permanently installed but are closed during times of floods to prevent floodwater intruding through railroad crossings or other features. These features are intentionally excluded from the crest profile data in NLD but should be included in hydraulic models as part of the levee crest. Locations of closure structures are available within the NLD as part of the structure’s Features tab. Without more detailed information, the crest of these closures should be interpolated from surrounding crest profile data.

#### Interior Drainage Features

Levee systems frequently include interior drainage features such as gravity drain culverts, pump stations, gated pipes, and other features. Information on these features can be gathered from the NLD or other available sources. These features should be included in the hydraulic model with the best available data or approximated when detailed information is not readily available. Interior drainage culverts often utilize flap-gates to prevent backflow from the river to the interior side of the levee. The flap-gate feature in HEC-RAS can be used to approximate these culverts’ behavior and the diagram shown in [Figure 1](#_Ref191457102) may be useful for determining whether the *no positive flow* or *no negative flow* flap gate option should be used. Pump stations can also significantly impact inundation near levees and where information is available, these pumps should be incorporated into the modeling. Detailed information regarding the pump stations may be unavailable, and in these cases, engineering judgement should be used to best approximate the impact of the pump station. Terrain modifications or other modeling techniques may be required to approximate some of the underground pipe networks which feed interior drainage systems.

## Levee Breach Data

Levee breaching will be performed using the hydraulic model and levees should be incorporated into the modeling with this use-case in mind. If breach locations are known, additional 2D cell refinement may be required in areas where breach floodwaters are expected. Mesh refinement behind levees should take care to balance model runtime objectives with mesh resolution and detail.

Additional detail regarding breach location, breach parameter and system response curve selection are covered in separate SOP materials.
