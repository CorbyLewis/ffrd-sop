# Job Aid 11—Levee System Response Development

This job aid describes the procedures to collect levee performance data, scope the levee breach modeling methodology, produce system response inputs, and communicate with the project team to incorporate levee breaches into the modeling system.

## Context

To incorporate levee performance into FFRD, one or more breach locations must be identified, system response probability (SRP) curve(s) must be estimated, and a breach scenario must be incorporated into the hydraulic and hydrologic models and cloud-compute inputs.

## Roles

The following key roles are required to incorporate levee breaches into an FFRD project.

### Levee Data Lead

The levee data lead is the USACE expert responsible for the development and delivery of SRP curves for levees. They also contribute to levee database refinement efforts. Additionally, they review and evaluate breach results from stochastic model simulations.

### Hydraulic Modeler

The hydraulic modeler is responsible for developing the model that fully accommodates levee breaches as scoped. Additionally, the hydraulic modeler confirms the levee control location (LCL) (initial overtopping locations) for each levee system based on modeled water surface profiles.

### Cloud Compute Lead

The cloud compute lead incorporates hydraulic and hydrologic models and SRP relationships into cloud compute for the project and ensures that results are shared with the team for review.

## Identifying Levees in the Study Area

The National Levee Database will be used as the authoritative listing of levee systems in the study basin, and every system will include at least one breach location for the FFRD project. NLD data refinement efforts that occur prior to the start of the FFRD project will ensure that the levee inventory is of high quality for project use. ## Levee Breach Quantity and Location Selection

Breach location selection and quantity is a critically important but challenging aspect of levee modeling. The location and number of breaches impacts the potential inundation area in the FFRD results.

The levee data lead determines the breach quantity. The breach quantity will be set by a combination of many factors including levee mileage, number of levee segments, and risk assessment data availability. The breach quantity can be adjusted based on engineering judgment. The breach location will follow a general hierarchy for most critical areas to define but will also rely on engineering judgment in some cases. Documentation of applied judgement is required (See Subsection [1.5.1](#levee-breach-quantity)).

The following subsections outline the procedures. Selecting breach quantity and location can be expected to be iterative with the HEC-RAS modeling team; particularly with identifying a LCL, which is the location on the levee system expected to experience overtopping first. The initial breach locations are needed by the hydraulic modelers ahead of the calibration phase. The calibration model runs may yield a different understanding of the LCL location that will need to be adjusted.

### Levee Breach Quantity

Breach quantity is a balance of the level of detail required to display a representative picture of flooding and the inundation characteristics of the levee. Long levee reaches, levee systems with steep terrain, and/or short hydrograph floods present a theoretical need to increase the number of breach locations to provide a more comprehensive breach inundation. For example, a breach location at the downstream end of a long levee system will not incur inundation to the upper reaches of the leveed area. If available information indicates this is the most likely scenario for levee performance to contribute to the system, it is acceptable. In the absence of risk assessment data, there is uncertainty in making the selection.

When using levee risk assessment information, the breach quantity will be scaled to accurately represent what is known about the levee and not mischaracterize levee performance by over manipulating the input data. For example, combining multiple system responses from different Potential failure modes (PFMs) and locating them at a LCL or frequently loaded location will increase the annual probability more than intended. Similarly, arbitrarily splitting input data to apply to a larger number of breach locations may result in levee performance mischaracterization by the modeler and increase computation demands. Best practice is to add breach locations for the most significant contributors to the expected annual probability of failure; essentially where the levee was expected to have performance concerns. This approach will best yield results consistent with how risk assessors intended. PFMs with a lower probability of occurrence compared to the governing PFMs are not expected to trigger in the modeling and contribute to FFRD products. Levee specific performance information of levee features including earthen embankments and floodwalls can be applied directly. The quantity of breaches will vary but typically will have a breach location for overtopping and at least one breach per feature (earthen embankment, floodwall, etc.). This approach increases the likelihood that FFRD results will be aligned with published risk assessment results.

When more detailed risk assessment information is not available, a minimum number of breaches should be implemented based on levee length to improve chances of characterizing inundation area from levee breaches.

Table 1 provides the recommended minimum number of levee breaches per levee length. The breach quantity can be adjusted based on engineering judgment. The actual number of breaches may increase based on the underlying risk assessment data supporting the evaluation. When siting multiple breach locations, breach locations should be spaced somewhat evenly and prioritize another breach location in the upstream increment of the levee system. For example, if a levee is recommended to have two breach locations and the LCL is located in the lower half of the levee, a representative, prior-to-overtopping location should be sought in the upper half of the levee system. Engineering judgment is required (See Section [1.5.2](#levee-breach-location)).

### Levee Breach Location

Once the target quantity of breaches per levee system is determined, the breaches can be located. Breach location selection is generally based on current USACE [Mapping, Modeling, and Consequences (MMC) Production Center’s Technical Manual for Levees](https://mmc.sec.usace.army.mil/levees-sop.html). Locations are identified using the following prioritized considerations:

1.  Breach location at the incipient overtopping location, also known as the LCL, as understood at the time of model setup including using initial without breach model runs. Note that this will likely require hydraulic modeling team input from initial calibration runs and be finalized iteratively.

<!-- -->

1.  Known areas of poor expected performance based upon risk assessment results.

2.  Locations of maximum levee height. Do not select areas with infrequent loading or those significantly lower than typical overtopping annual exceedance probability (OT AEP). Refer to loading function, as available. Taller levee sections have more potential susceptibility to various PFMs and would yield significant breach inflows.

3.  Location in the upstream third of the levee system.

4.  High population or structure density areas adjacent to the levee.

While this list is provided in priority order, combinations of factors may present a stronger case than a rigid hierarchy. For example, a representative breach location adjacent to a population center in the upstream third of the levee may be considered a more critical location than a maximum height levee at the downstream end of the levee. FFRD steering committee guidance stated that FFRD flood hazard data is important for land use planning as it is for current structures; therefore, locating breaches to inform flooding to populated areas is a minor consideration overshadowed by the need to model most likely overtopping and breach locations. Engineering judgment is required. Location selection rationale must be documented.

#### Multiple Breach Locations for a Levee System

The SRP curves should be as representative of each unique breach location identified. For example, a single breach location at the LCL will have a simplified assumption that all critical prior-to-overtopping performance is combined and applies at the same location most likely to overtop. For levee systems with multiple breaches assigned, breaches will be assigned performance data that reflects the reason for selecting the location. For example, a levee system that has a LCL and two likely breach locations will have the breach locations represent the system response for those failure modes and the LCL represents the remainder of the performance evaluated during the risk assessment.

## System Response Probability Curve Development

An SRP curve represents the conditional probability of breach for a levee location. Figure 2 shows an example of a levee SRP curve plotted in terms of elevation versus probability of breach. The system response curve, a cumulative distribution function, provides a conditional probability of breach for prior-to-overtopping and overtopping events. The SRP curve begins at the origin (0,0) and increases monotonically.

The relationship of system response and annual probability is important to understand prior to developing SRP curves. Figure 3 provides an illustrative example of the area under an SRP curve and what that equates to in an annualized value. The curve is simplified to show p(f) of 1.0 at flood frequencies (AEP) exceeding the crest. The x-axis is the AEP of a flood loading event and the y-axis is the conditioned probability of levee breach. In the figure, the area under the entire system response curve is the annual probability of inundation (API). The area under the portion of the curve prior to overtopping AEP is referred to as the annual probability of breach prior to overtopping (APF<sub>PTOT</sub>) and the remaining portion of the curve represents the overtopping contribution to the API.

``` math
{APF\ }_{TOT} = \ {APF}_{PTOT}\  + \ {API}_{OT}
```

A few key terms are important to define to build the understanding of how SRP curves are developed:

**PFM:** A PFM is a mechanism that once initiated potentially could progress to breach of a levee system or inundation of the leveed area.

**Probability of Failure (p(f) or p(fail)):** The probability of levee breach conditioned upon a given water surface elevation, percent levee height, or flood frequency.

**Annual Probability of Failure Prior to Overtopping (APF<sub>PTOT</sub>):** The annual probability of failure (APF) represents the likelihood, in any given year, that a levee will breach considering the full range of potential loads to which it could be subjected (LST Technical Reference Manual). The APF<sub>PTOT</sub> is a summation of only the likelihood of breach for loading below the levee crest.

**API:** The API is the APF<sub>PTOT</sub> plus the annual probability of overtopping with and without breach.

**OT AEP:** The probability that levee overtopping will occur in any given year considering the full range of possible annual floods.

SRP curve development utilizes risk assessment information and/or USACE levee loading and incident data. The development process is dependent on if the levee has been assessed with a risk assessment or not. Further, if the levee does have a risk assessment, the type of risk assessment informs SRP curve development.

QRA and LST risk assessments produce SRP curves for each PFM evaluated. FFRD SRP curves developed from these sources are combined, distributed, and/or down-scaled appropriately for the levee breach location scheme.

Levee systems without levee-specific performance data will be assigned a system response of an average levee based on the USACE LLID. FFRD SRPs developed are then combined, distributed, and/or down-scaled appropriately for the levee breach location scheme.

### Multiple Overtopping Locations on the Same System

In consideration of multiple levee breach locations, every breach location assigned will include an overtopping system response. This will model a realistic scenario in which all potential areas of overtopping have a likelihood of breaching from overtopping. USACE subject matter experts (SMEs) concluded this approach creates a situation for the model to perform its own competing risk calculation, minimizing the length effect concern. Theoretically, as the initial overtopping location begins to overtop and possibly breach, the river stage at the other breach locations is less likely to achieve an elevation needed to trigger a breach. Thus, each of the individual breach locations of a particular levee system are competing against each other for the chance to breach. Once one is successful, the probability of another occurring drops so as not to introduce length effect issues of the probability of failure approach.

### Starting with a Quantitative Risk Assessment

From a QRA report, the probability of failure for a given percent loading, flow, or event AEP is typically available directly, but this information needs to be manually collected and then assigned to the breach location setup previously determined. Breach location assignment may involve combination of one or more PFMs and the addition of expected overtopping performance. If loading function information is available at multiple locations and varies significantly, the PFM specific system response should be converted to an AEP versus p(f) format before combining.

Combination of likelihood of breach for multiple PFMs into a single breach location is accomplished with simple addition using the formula below. This calculation assumes the PFMs are mutually exclusive. The combination also assumes that the frequency of flood loading is common at a given percent loading in order to arrive at the intended APF for the system. For example, combination of two SRP curves for different PFMs at a 50 percent loading must mean that the 50 percent loading at each of the locations corresponds to approximately the same frequency of flood loading (i.e., 50 percent loading at both locations is equivalent to a 1 in 50 AEP).

``` math
{P(fail)}_{Total\ } = \ {\sum p(fail)}_{i}
```

### Starting with a Semi-Qualitative Risk Assessment

In most cases, a semi-quantitative risk assessment will not include SRP curves for each PFM. The output users will be looking for from SQRA will be loading functions and the evaluated PFMs and their respective elicited APF with intervention. The general process for developing SRP curves from this data is to set the APF as a target and develop a SRP curve where the area under the curve match that APF target. The shape of the curve is influenced by log normal shape parameter (sigma) that are from USACE LLID. USACE maintains a spreadsheet for this computation. The detailed steps are:

1.  Evaluate hydraulics and hydrologic data in the report. Locate and enter one or more flood loading functions required to represent the flood loading for the PFMs. Input data needed: Stage and AEP for each loading curve. Enter the incipient overtopping point to document the cutoff point for APF calculation.

<!-- -->

11. Locate and enter the APF with intervention for each elicited PFM as the target APF for the calculated SRP for each PFM. All credible breach modes from the SQRA report will be used to ensure that the total APF for the system is considered. Assign each PFM to a standard PFM type to associate the LLID shape parameters.

12. Perform log-linear interpolation of each percent loading function to a standardized set of nodes to normalize all PFM entries to a common set of AEP range so SRPs can eventually be combined.

13. Set an iterative computation of SRP at each AEP node by varying shape parameter (σ), *sigma ln*, until the computer APF of the created SRP curve matches the target APF from the SQRA report.

14. Repeat for all credible PFMs.

15. Combine all or portions of the calculated SRP to match the desired distribution of PFMs at the breach locations. A system represented by only a single breach location will sum all calculated PFM SRP curves. A levee system represented by multiple will use specific PFM SRP curves directly and sum the remainder at the LCL, for example.

[Figure 5](#_Ref216706655) shows the calculation spreadsheet setup with orange cells indicating entry. Fail mode is selected from a set dropdown list. User selects the loading function that applies to this PFM. User enters the elicited PFM APF with intervention from the SQRA report. Finally, an iterative approach is used to change sigma ln until the computed APF matches the target APF. The prior to overtopping SRP curve is now created for this PFM. The expansion of percent levee height is a log-linear interpolation of the input flood loading curve. The calculation of SRP is done as a cumulative density function with a log-normal distribution for the given percent levee height ordinate with mean (μ) and standard deviation (σ). Finally, the computation of APF is computed as the area under the SRP curve using a Riemann sum trapezoidal approximation:

``` math
APF = \sum_{}^{}{\left( x_{i} - x_{i + 1} \right) \times \left( y_{i} + y_{i + 1} \right)} \div 2
```

### Starting with a Screening Level Risk Assessment

Screening level risk assessment data is available in the LST. Once a multi-disciplinary team completes a screening, the LST maintains a database information on levee performance (PFMs), representative breach locations assigned to those PFMs, and representative loading functions assigned to those PFMs. The LST uses a standard set of PFMs that are assessed depending on the features of the levee (e.g., earthen embankment, floodwall, and/or closure structure). Overtopping with breach for the incipient overtopping location is also assessed.

### Starting with a Screening Level Risk Assessment (LST 1.0)

The most prevalent levee performance on levees in the United States comes from the first version of the LST (LST 1.0) with a screening level risk assessment completed for the entire USACE portfolio.

The input data needed to develop SRP curves from this data set is the LST 1.0 database export file from October 2023. The file contains levee toe and crest loading frequency (AEP) and performance indexes for each PFM. The user should understand that it is typical that only a single generalized loading curve representing the most frequent loading toe AEP and the incipient overtopping frequency are available. A segment may have a separate loading function for embankments, floodwalls, and/or closure structures if those features are present. Further, the levee performance was based on a relative risk Bayesian updating approach. Conversions of the performance data are included to correct the performance estimates to expected magnitudes and shape parameters from the USACE Levee Loading and Incident data as well as corrections to account for intervention. ### System Response Allocation, Combination, and Formatting

SRPs representing performance for a specific PFM or levee segment are then combined to allocate system response to the defined breach location scheme. If the breach scheme identifies the need for a prior-to-overtopping breach location and the risk assessment source has identified a critical flaw at this specific location, the full system response for that breach mode is applied. If no critical location is identified, the combined performance modes evaluated in the risk assessment are down-scaled and applied evenly to all breach locations. The combination computation applies De-Morgan’s rule for statistically independent and not mutually exclusive PFMs when the risk assessment data is unadjusted ([Table 8](#_Ref206504229)).

$`Probability\ of\ Failure = 1 - \left( 1 - p(fail) \right)^{N}`$

<span id="_Ref206504229" class="anchor"></span>Table 8. Summary of Adjusted and Unadjusted System Response by Risk Assessment Type

| Risk Assessment Type | Adjusted or Unadjusted | Combination Method |
|----------------------|:----------------------:|:------------------:|
| QRA                  |       Unadjusted       |  De-Morgan’s Rule  |
| SQRA                 |       Unadjusted       |  De-Morgan’s Rule  |
| LST (Current)        |        Adjusted        |     Summation      |
| LST (v1.0)           |        Adjusted        |     Summation      |

When levee SRPs need to be distributed to multiple identified breach locations, an adjustment to De-Morgan’s rule is applied to the conditional probability of failure for the system (P(fail)) to statistically down-scale the system response the number of breach locations (N<sub>Locations</sub>).

``` math
Probability\ of\ Failure = 1 - \left( 1 - P(fail) \right)^{{1/N}_{Locations}}
```

Once the SRP is developed for the breach location scheme, the data is converted to the correct stage range from levee toe elevation to crest elevation. The overtopping response is added using De-Morgan’s rule to the prior-to-overtopping system response, assuming that prior-to-overtopping at 100 percent of the levee height remains constant. Finally, if the probability of breach has not reached 1.0, the SRP curve is completed by interpolating from the overtopping response.

### Combining Overtopping performance to Prior to Overtopping

Risk assessments typically document overtopping differently and include a direct SRP curve that expresses probability of failure as a function of depth over the crest of the levee at that location. Therefore, this portion of the SRP curve will be added after the SRP curve is in the elevation versus p(f) format.

The elicited overtopping performance typically has a maximum range (i.e., only elicited to 2 feet over the levee crest height). For the purposes of FFRD, the overtopping performance will be extrapolated to 99.9 percent chance of breach and then given a straight line 100 feet above the levee crest elevation to ensure that the sampled elevation range exceeds the potential FFRD hydraulic model elevation.

The final step when combining prior to overtopping and overtopping portions of the SRP is to consider that prior to overtopping PFMs have a non-zero chance of breach, even during overtopping. For example, a levee may still breach from internal erosion during overtopping so it will be combined with likelihood of overtopping with breach; however, tailwater conditions as the leveed area fills ultimately cause the internal erosion SRP to plateau. To account for this, the prior to overtopping probability of failure is held as a constant during the addition of the overtopping performance probability of failure. In reality, this is not a constant, but this approach is used as a simplification to not ignore this contribution.

## Format for Levee System Response Probability Curve Delivery

The SRP curves are compiled, quality control reviewed, and then uploaded to Model Library as a single .CSV file following the format as [Figure 11](#_Ref216707784) shows. The file includes the 2D connection name in HEC-RAS model to link the system response trigger elevation sampled from the system response curve to the water surface modeled in HEC-RAS. The remaining common data fields are data identifiers including the National Levee Data system and segment ID numbers, and the basis for the system response curve (i.e., risk assessment source, potential failure mode, or default performance). The system response includes prior to overtopping and overtopping performance.

The provided SRPs formatted as .CSV files will be used as an input for the cloud-compute actions that sample breach trigger elevations for the stochastic simulations.

A template for use in creation of the required SRP format for delivery in included in Appendix C, named *system-response*. Note that no commas are allowed in the header information fields.

The levee SRP .CSV file should be initially delivered to the project directory in the ‘\[basin-name\]\>calibration\>system-response’ path.

## Review Evaluation of Breach Results

A single realization of stochastic events is simulated by cloud compute during the conformance phase. Breach results from the conformance phase must be reviewed by the dam data lead to ensure that breach trigger elevations are being sampled appropriately and the number of breach events that occur at each breach location falls within the expected range. Similarly, the multi-realization results from the projection phase must be reviewed and evaluated.
