# Job Aid 4—Observed Gage Data Preparation

An authoritative set of observed data will be collected during the Data Preparation Phase and shared amongst the modeling team. Subsequent modeling efforts must make use of this authoritative set of observed data in order to avoid using conflicting data. Data that is gathered must also be adjusted prior to use to adhere to vertical datum and time zone requirements (as established in SOP Volume II, Section 3).

The authoritative dataset will be saved to the central *basin-data* directory for the project.

If other data becomes available during later phases (e.g., Calibration, Conformance, etc.), it must be incorporated into the authoritative set of data and shared with the entire modeling team. Any updates to the authoritative dataset that are must be coordinated with the technical lead.

This document describes the types and sources of data, data processing tools, and typical modifications that are required for use within FFRD applications.

## Types of data to be collected

Several types of data should be collected during an FFRD application. This includes:

- Flow

- Stage

- Stage-Discharge Relationships

- Stream Gage Field Measurement Dataset

- High water marks

- Flood imagery-satellite or airborne

- Flood Photography-ground photography

Additionally, data should be collected at numerous temporal resolutions including (but not limited to):

- Instantaneous (e.g., 5-, 10-,15, 30-, and/or 60-minutes)

- Sub-daily (e.g., 3-, 6-, and/or 12-hours)

- Daily

- Annual Maximums

## Data Sources

The United States Geological Survey (USGS) collects water resources data at over 10,000 active stream gages. This data is disseminated through websites like the [USGS Water Data for the Nation site](https://waterdata.usgs.gov/nwis/). Additional information including historical observations, statistics, instantaneous peak flow/gage height, and field measurements are also available. In most FFRD applications, data collected and distributed by the USGS will constitute the vast majority of pertinent data. However, there are numerous additional sources of water resources data that may be pertinent to FFRD applications including (but not limited to):

- <https://tidesandcurrents.noaa.gov/>

- <https://www.usbr.gov/pn/hydromet/index.html>

- <http://cdec.water.ca.gov/cdecstation2/>

- <https://txwaterdatahub.org/dataset/highwater-marks-inventory>

- <https://water.usace.army.mil/>

- <https://stn.wim.usgs.gov/FEV/> (high water mark data)

- <https://www.usgs.gov/centers/eros> (EROS satellite imagery depot)

An excellent summarization of pertinent data sources that are commonly utilized within USACE dam and levee safety studies can be found here: [Data Sources for Estimating Hydrologic Hazards for Semi-Quantitative Risk Assessments](https://publibrary.sec.usace.army.mil/resource?title=Data%20Sources%20for%20Estimating%20Hydrologic%20Hazards%20for%20Semi-Quantitative%20Risk&documentId=1d66bd71-3a95-4248-f96c-93ad4e3ca81a).

## Vertical Datums

Commonly, historical data will be reported in a local/project-specific vertical datum or the National Geodetic Vertical Datum of 1929 (NGVD 29). As established in SOP Volume II, Section 3, all stage and/or elevation data used within FFRD applications must be refenced to the North American Vertical Datum of 1988 (NAVD 88). Utilities such as the [NGS Coordinate Conversion and Transformation Tool](https://geodesy.noaa.gov/NCAT/) or [NOAA’s VDatum](https://vdatum.noaa.gov/) can be utilized to convert stages and/or elevations to NAVD 88.

Conversions from local project datums will also be clearly defined during the scoping and data preparation phase when they are available.

## Acquisition of Stream Gage Data

For all stream gages within the modeling domain, download and/or ingest the following data using HEC-SSP or HEC-DSSVue:

- Instantaneous and/or sub-daily stage and flow data

- Daily average flow data

- Annual maximum series of flow

- Annual maximum series of stage

- Stream gage field measurement paired dataset (flow and stage)

- Published stage-discharge relationships

Examples demonstrating these capabilities can be found here: [Create a new HEC-SSP Study, Import Data, and Visualize Multiple Data Sets](https://www.hec.usace.army.mil/confluence/sspdocs/ssptutorialsguides/introduction-to-hec-ssp/task-1-create-a-new-hec-ssp-study-import-data-and-visualize-multiple-data-sets).

Additionally, several commonly available scripting languages (i.e., R and Python programming languages) have libraries that can be utilized to download this data as well. Example of these libraries include:

- <https://doi-usgs.github.io/dataRetrieval/>

- <https://github.com/DOI-USGS/dataretrieval-python>

## Extracting N-Day Duration Time Series

For all stream gages within the modeling domain, calculate n-day duration flow time series (e.g., 1-, 2-, and 3-day durations) using HEC-SSP or HEC-DSSVue. ## Quality Assurance/Quality Control of Time Series Data

Performing quality control on observed data is a critical step in all FFRD applications as the use of incorrect or poor-quality observed data may result in inaccurate results. An excellent summarization of quality assurance (QA)/quality control (QC) processes which must be undertaken during flood hazard studies is contained within the [Hydrologic Hazard Methodology for Semi-Quantitative Risk Assessments: An Inflow Volume-Based Approach to Estimating Stage-Frequency Curves for Dams](https://publibrary.sec.usace.army.mil/resource?title=Hydrologic%20Hazard%20Methodology%20for%20Semi-Quantitative%20Risk%20Assessments:%20An%20Inflow%20Volume-Based%20Approach%20to%20Estimating%20Stage-Frequency%20Curves%20for%20Dams&documentId=87363a2a-8dd9-4596-991e-2f9863815c7e). The following text was summarized from that document.

In cases where systematic data is obtained for streamflow gages, information on manual measurements of streamflow and gage height, including indirect measurement, can be obtained from the USGS at <http://waterdata.usgs.gov/nwis/measurements>. These measurements can then be used to develop streamflow rating curves, supplement and/or verify the accuracy of the automatically recorded estimates at gages, as well as to compute streamflow based on gage height. These are valuable for flood frequency studies to aid hydrologists in understanding how the largest flood estimates are made (such as indirect measurements), and in estimating uncertainty. See details on streamflow measurement and computation and discharge ratings from the USGS at <https://water.usgs.gov/osw/furnished_records/technical_procedures.html>. At most large dams, reservoir inflow is indirectly calculated using measurements of reservoir storage (via stage) and outflow (via rating curves or gage measurements). The water management entity that owns/operates the reservoir in question typically utilizes QA/QC steps on the calculated reservoir inflows, so the data may be used as provided after a cursory review is made to check for any obvious errors.

In addition to potential measurement errors, many datasets have missing data, duplicate values, erroneous data, or other issues that can impact the inflow volume- and stage-frequency assessments. Plotting the flow and stage time series data can assist with identifying any potential errors.

Chapter II-1, *Reservoir and River Stage Exceedance Probabilities*, from Best Practices (U.S. Bureau of Reclamation and U.S. Army Corps of Engineers, 2015) provides a useful example data set showing daily average reservoir stages for a dam as presented in Figure 1. The plot reveals several potential data quality issues. The time periods associated with the initial reservoir filling, the dam safety emergency, and the pool restriction for interim risk reduction may not be representative of normal operation. Some of the data also appears to be missing and incorrect based on a visual inspection of the plot. Missing data should be filled and flawed data should be corrected or removed, as deemed appropriate.

## Produce Probabilistic Summaries

Probabilistic summaries of time series data will be used to ascertain the accuracy of the modeling tools and techniques within Calibration, Conformance, and Production Phases.

### Determine Regulated/Unregulated Time Windows

Prior to extracting any probabilistic results, homogenous time windows for each stream gage must be identified. This must be done in order to avoid violating the independent and identically distributed assumption and align with FFRD modeled outputs. When there is a known source of regulation upstream of a location of interest, but the extent of the impacts is uncertain, further investigation is required. An excellent summarization of steps that can be taken to ascertain the degree of regulation is contained within the technical report titled, [Estimating Flood Hazard for Dams and Levees with Upstream Regulation](https://iwrlibrary.sec.usace.army.mil/resource?title=Flood%20Hazard%20Methodology%20for%20Semi-Quantitative%20Risk%20Assessment:%20Estimating%20Flood%20Hazard%20for%20Dams%20and%20Levees%20with%20Upstream%20Regulation&documentId=a645ddc5-83d6-4781-9a73-6a6c1e3d128e).

For all locations, consider changes in land use, etc. that may impact the homogeneity of the observed data. Also, for locations that are subject to upstream regulation, consider changes in regulation from upstream storage and diversion projects that may impact the homogeneity of the time series (i.e., not all periods of regulation should be considered significant).

### Annual Maximum Series

Commonly, annual maximum series (AMS) of flow and stage/elevation will be available at most (if not all) USGS gages within the modeling domain. However, these AMS are typically indicative of an instantaneous peak duration (i.e., 1-hour). While the instantaneous peak duration is adequate for use when comparing observed and computed stage/elevation data, additional durations for flow data are needed. Thus, for all stream gages within the modeling domain, flow AMS must be extracted from the observed data for 1-day, 2-days, and 3-days.

### Plotting Positions and Confidence Limits

For all stream gages within the modeling domain, 5- and 95-percent confidence limits (i.e., 90 percent confidence interval) must be computed for both flow and stage/elevation for homogenous time periods.

Confidence intervals for all homogenous flow AMS should be computed using one of the following approaches:

- For unregulated locations and time periods, use either multiple Bulletin 17 (i.e., one analysis per duration) and/or a single volume frequency analysis to compute confidence intervals using Bulletin 17C procedures. Examples demonstrating these capabilities can be found here: [Bulletin 17C \| Expected Moments Algorithm Examples](https://www.hec.usace.army.mil/confluence/sspdocs/sspexamples/latest/bulletin-17c-examples/bulletin-17c-expected-moments-algorithm-examples).

- For regulated locations and time periods, use a general frequency analysis to graphically fit an empirical distribution and estimate confidence intervals using order statistics. Examples demonstrating these capabilities can be found here:

  - [Create and Parameterize a General Frequency Analysis](https://www.hec.usace.army.mil/confluence/sspdocs/ssptutorialsguides/introduction-to-hec-ssp/task-3-create-and-parameterize-a-general-frequency-analysis)

  - [General Frequency – Graphical Analysis](https://www.hec.usace.army.mil/confluence/sspdocs/sspexamples/latest/general-frequency-analysis-examples/example-9-general-frequency-graphical-analysis)

Confidence intervals for all homogenous stage/elevation AMS should be estimated by first graphically fitting an empirical distribution and then computing using order statistics. Examples demonstrating these capabilities can be found here:

- [Create and Parameterize a General Frequency Analysis](https://www.hec.usace.army.mil/confluence/sspdocs/ssptutorialsguides/introduction-to-hec-ssp/task-3-create-and-parameterize-a-general-frequency-analysis)

- [Example 7. Analyzing Stage Data](https://www.hec.usace.army.mil/confluence/sspdocs/sspexamples/latest/general-frequency-analysis-examples/example-7-analyzing-stage-data)

## Format for Delivery

Observed data should be provided in HEC DSS format for efficient usage by subsequent modeling applications. Probabilistic results must be provided in both .DSS and Comma-Separated Values (.csv) format for use within the Calibration, Conformance, and Production Phases. The prescribed naming convention laid out within the FFRD SOP must be utilized.

Authoritative observed data will be posted to the *basin-data/observed-data* central directory for the project at the end of the Scoping and Data Preparation phase.

## Documentation of Data Collection Efforts

All data collection efforts must be summarized and provided to the modeling team and included in the report for the Scoping and Data Preparation Phase.

Entries for observed data will be included in the Data Manifest.
